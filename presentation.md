## Slide 8: Database Design & Tenant Isolation
**Presenter:** Shehan
**Visuals to Include:** A simplified ERD showing the `Organization` table mapping to `Employee`, `Attendance`, and `LeaveRequest` via `organization_uuid`.

**Slide Content:**
* **Single-Database, Multi-Tenant Design:** All client organizations share the same database structure.
* **Tenant Isolation:** Every major table includes an `organization_uuid` foreign key.
* **Security Filter:** Spring Data JPA queries strictly filter by `organization_uuid` derived from the user's JWT token.
* **Scalability:** Very easy to maintain and deploy compared to Database-per-Tenant models.

**English Script:**
"To achieve SaaS multi-tenancy affordably, we use a 'Single-Database, Shared Schema' design. To prevent data leaks between companies, we enforce strict Tenant Isolation. Every major table has an `organization_uuid` foreign key. The backend extracts this UUID dynamically from the user's JWT token and filters every SQL query, guaranteeing complete data privacy."

**Sinhala Script:**
"අපි පාවිච්චි කළේ 'Single-Database' කියන ක්‍රමය. ඒ කියන්නේ ඔක්කොමලාගේ data තියෙන්නේ එකම database එකක් ඇතුලේ. ඒක වෙන් කරන්න අපි හැම table එකකටම 'organization_uuid' කියලා foreign key එකක් දුන්නා. කෙනෙක් ලොග් වුනාම එයාගේ JWT token එකෙන් ඒ UUID එක අරන් query එකටම දාලා ෆිල්ටර් කරන නිසා එක කොම්පැණියක විස්තර තව කොම්පැණියකට කීයටවත් පේන්නේ නෑ."

**English Script:**
"Here are some core data flows. When an organization pays via PayHere, a webhook triggers our backend to auto-activate their paid modules.
